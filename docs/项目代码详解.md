# Guangzhou-Metro-Monitor-System 代码详解

> 基于 CesiumJS 的广州地铁 1 号线综合监控系统 —— 完整代码说明
> 适用对象：自己回头看代码时的备忘录

---

## 一、项目概览

这是一个纯前端的 3D 地铁监控仪表盘。所有数据来自本地 JSON 和 GeoJSON 文件，不涉及后端服务。

**3D 地球场景仅展示地铁线路走向和站点位置，不包含人口数据、建筑模型或其他地理信息主题。**

### 页面布局（12 列 CSS Grid）

```
+------------------+------------------+------------------+
|  左面板 (3列)     |  中栏地图 (6列)   |  右面板 (3列)     |
|                   |                   |                   |
|  实时客流排行      |   Cesium 3D 地球  |  站点详情          |
|  系统告警         |   1 号线高亮      |  1 号线运行状态    |
|                   |                   |                   |
|                   +-------------------+                   |
|                   |   底部指标卡       |                   |
|                   | 准点率/状态/在线   |                   |
+-------------------+-------------------+-------------------+
```

### 技术栈

| 技术 | 作用 |
|------|------|
| CesiumJS 1.137 | 3D 地球渲染 |
| Tailwind CSS | 布局（Grid/Flex） |
| 原生 JavaScript | 所有逻辑 |
| GeoJSON | 地理数据 |
| JSON | 业务数据 |

## 三、数据文件详解（如何替换）

### 3.1 data/stations.json —— 站点属性

这是站点详情面板和客流排行的数据源。

```json
{
  "stations": [
    {
      "id": 1,
      "name": "广州东站",
      "transfers": "换乘3号线,11号线,18号线",
      "isTransfer": "yes",
      "flow": 28500
    }
  ]
}
```

**各字段含义：**

| 字段 | 说明 | 替换方式 |
|------|------|----------|
| `name` | 站名 | 必须与 `GZLine1_Station.geojson` 中的 `name` 一致 |
| `transfers` | 换乘文字 | 直接修改字符串 |
| `isTransfer` | 是否换乘站 | "yes" 或 "no" |
| `flow` | 客流量（数字） | 修改数字即可改变排行和详情显示 |

**替换方法：** 修改 `flow` 字段的值即可改变客流排行和详情面板中的客流量数据。站名必须和 `GZLine1_Station.geojson` 中的 `name` 字段一一对应，否则点击站点找不到匹配数据。

### 3.2 data/status.json —— 运行状态

控制右侧状态灯和左侧告警面板。

```json
{
  "status": [
    {
      "name": "广州东站",
      "status": "normal",
      "message": "",
      "abnormaltime": ""
    }
  ]
}
```

**各字段含义：**

| 字段 | 说明 |
|------|------|
| `name` | 站名，必须与 stations.json 一致 |
| `status` | "normal" = 正常, "abnormal" = 异常 |
| `message` | 异常时显示的消息文字 |
| `abnormaltime` | 异常发生时间 |

**替换方法：** 把任意站点的 `status` 改为 `"abnormal"`，并填写 `message` 和 `abnormaltime`，就能看到：
- 右侧该站点状态灯变红
- 左侧告警面板出现该站点
- 底部系统状态变为"异常待处理"
- 全屏红色闪烁告警

### 3.3 data/dashboard.json —— 仪表盘数据

```json
{
  "onTimeRate": 99.8,
  "onlineTrains": 482,
  "systemStatus": "NORMAL"
}
```

### 3.4 GZLine1/GZLine1.geojson —— 线路路径

这是 1 号线的地理坐标数据，格式为标准 GeoJSON LineString。你可以用 ArcGIS 或 QGIS 导出新的线路替换它。

### 3.5 GZLine1/GZLine1_Station.geojson —— 站点坐标

格式为 GeoJSON Point 集合，每个点包含站点名称：

```json
{
  "type": "GeometryCollection",
  "geometries": [
    {
      "type": "Point",
      "coordinates": [113.3191, 23.1527],
      "name": "广州东站"
    }
  ]
}
```

**stations.json 和 GZLine1_Station.geojson 通过 name 字段关联**，两个文件的站名必须一致。

### 3.6 数据依赖关系图

```
GZLine1.geojson ──────────→ Cesium 线路渲染
                                   │
GZLine1_Station.geojson ──→ 站点坐标
       │                         │
       │  (name 字段关联)         │
       ▼                         │
stations.json ────┬──→ 站点详情面板 │
                  ├──→ 客流排行面板  │
                  │                  │
status.json ──────┬──→ 线路状态灯    │
                  ├──→ 告警面板     │
                  └──→ 异常闪烁     │
                                   │
dashboard.json ───→ 底部指标卡     │
                                   │
apikey.txt ───────→ Cesium Token  │
                                   ▼
                             3D 地球场景
```

### 4.3 脚本 1：Cesium 地图初始化（`<script type="module">` ）

这是最核心的脚本块，执行整个 3D 场景的构建和交互逻辑。

#### 执行流程

```
1. 读取 apikey.txt → 设置 Cesium Ion Token
2. 创建 Viewer → 关闭所有默认控件和版权信息
3. 禁用默认点击事件 → 替换为自定义交互
4. 相机飞行 → 定位到广州市区上空
5. 加载 GZLine1.geojson → 渲染 1 号线路径
6. 应用流光材质 → 线路动画
7. 加载 GZLine1_Station.geojson → 获取 16 站坐标
8. 加载 data/stations.json → 获取站点属性
9. 调用 addStationPoint() → 在地图放置 16 个站点
10. 注册点击事件监听
```

#### addStationPoint() 函数

```javascript
function addStationPoint(res) {
    for (let station of res.geometries) {
        // 从 stations.json 查找对应站点的属性
        const stationInfo = stationsData.stations.find(s => s.name === station.name);

        const pos = Cesium.Cartesian3.fromDegrees(
            station.coordinates[0], station.coordinates[1]
        );

        const entity = viewer.entities.add({
            name: 'station',           // 打标签，用于点击时识别
            position: pos,
            billboard: {
                image: './GZmetro.jpg',
                width: 20, height: 20,
                disableDepthTestDistance: Number.POSITIVE_INFINITY, // 始终显示在最前
            },
            label: {
                text: station.name,
                font: '16px 微软雅黑, sans-serif',
                fillColor: Cesium.Color.WHITE,
                outlineColor: Cesium.Color.BLACK,
                outlineWidth: 2,
                verticalOrigin: Cesium.VerticalOrigin.BOTTOM,
                pixelOffset: new Cesium.Cartesian2(0, -30), // 标签在图标上方 30px
            },
        });

        // 重要：额外挂载站点数据
        // Cesium Entity 构造器只处理内置属性，自定义属性 stationData 会被丢弃
        entity.stationData = stationInfo;
    }
}
```

**关键点：** `stationData` 必须在 entity 创建之后单独设置。如果放在构造参数里传进去，Cesium 的 Entity 构造器会忽略不识别的属性。

#### 点击事件逻辑

```javascript
const handler = new Cesium.ScreenSpaceEventHandler(viewer.scene.canvas);
handler.setInputAction((click) => {
    const pickedObject = viewer.scene.pick(click.position);

    if (Cesium.defined(pickedObject) && Cesium.defined(pickedObject.id)) {
        const entity = pickedObject.id;

        if (entity.name === 'station' && entity.stationData) {
            const data = entity.stationData;

            // 显示详情面板
            document.body.classList.add('show-detail');

            // 填充数据
            document.getElementById('detail-title').textContent = data.name;
            document.getElementById('d-name').textContent = data.name;
            document.getElementById('d-transfers').textContent = data.transfers;
            document.getElementById('d-isTransfer').textContent =
                data.isTransfer === 'yes' ? '是' : '否';
            document.getElementById('d-flow').textContent =
                data.flow.toLocaleString() + ' 人';

            highlightStation(entity);
        }
    }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK);
```

**拾取原理：** `viewer.scene.pick()` 检测鼠标点击位置下的第一个对象。如果是 Entity 的 Billboard 或 Label，返回的对象中 `id` 字段指向该 Entity。

#### 高亮逻辑

```javascript
let selectedEntity = null;

function highlightStation(entity) {
    // 恢复上一次选中的站点
    if (selectedEntity) {
        selectedEntity.billboard.color = Cesium.Color.WHITE;
        selectedEntity.billboard.scale = 1.0;
    }

    // 高亮新站点
    selectedEntity = entity;
    if (entity && entity.billboard) {
        entity.billboard.color = Cesium.Color.YELLOW;
        entity.billboard.scale = 1.5;
    }
}
```

## 六、流光动画材质（js/PolylineTrailLinkMaterialProperty.js）

### 作用

让 1 号线的黄色线条产生光斑沿线路移动的动画效果。

### 原理

在 GPU 上运行的 GLSL 着色器代码：

```
对线路上的每个像素：
  算该像素在线路上的位置（0 = 起点, 1 = 终点）
  算当前光斑中心的位置（随时间推进）
  测量该像素到光斑中心的距离
  越近越亮（高斯函数衰减）
  每帧把光斑中心往前移一点 → 产生移动感
  光斑走到终点后回到起点 → 无缝循环
```

### 使用方式

在 `index.html` 中：

```javascript
const trailMaterial = new PolylineTrailLinkMaterialProperty(
    Cesium.Color.fromCssColorString('#F3D03E'),  // 黄色
    120                                            // 移动速度
);
entity.polyline.material = trailMaterial;
```

### 替换颜色

只需改 `fromCssColorString()` 中的颜色值即可，不需要动着色器代码。

## 七、数据替换指南（接口替换）

这个项目所有的"接口"就是 fetch 本地 JSON 文件。如果要接入真实后端，需要改 fetch 的 URL。

### 替换站点数据

```
index.html 中需要修改的位置：

1. Cesium 脚本块（加载 GZLine1.geojson 和 stations.json）
   → 改成其他城市的 GeoJSON 和站点数据

2. 客流排行脚本块（加载 stations.json）
   → 改成后端 API 地址

3. 线路状态脚本块（加载 status.json）
   → 改成后端实时推送或轮询地址

4. 仪表盘脚本块（加载 dashboard.json）
   → 改成后端 API 地址
```

### 扩展另一条线路

如果要加 2 号线：

1. 准备 2 号线的 `GZLine2.geojson` 和 `GZLine2_Station.geojson`
2. 在 `stations.json` 新增站点（或新建文件）
3. 在 `status.json` 新增站点状态（或新建文件）
4. 复制 Cesium 加载和渲染的代码，换成 2 号线的文件
5. 调整右侧面板的标题和站点列表

### 全局替换步骤

```
1. 更换 GZLine1/*.geojson                → 其他城市/线路的坐标数据
2. 修改 data/stations.json               → 对应站点的名称和客流
3. 修改 data/status.json                 → 对应站点的运行状态
4. 修改 index.html 中 lineStations 数组   → 对应站点的顺序列表
5. 替换 GZmetro.jpg                      → 新的站点图标
6. 修改 tailwind.config 中的颜色          → 换肤
```

## 九、常见问题排查

**Q: 直接用浏览器打开 index.html 为什么白屏？**
A: CesiumJS 需要跨域加载资源，必须通过本地服务器访问（VS Code Live Server 或 `python -m http.server`）。

**Q: 点击站点没反应？**
A: 检查 `stations.json` 和 `GZLine1_Station.geojson` 中的站名是否一致。点击事件通过 `name` 字段匹配站点数据。

**Q: 站点数据不显示？**
A: 检查 `apikey.txt` 中的 Cesium Ion Token 是否有效。

**Q: 如何新增站点？**
A: 同时修改 3 个地方：
1. `GZLine1_Station.geojson` 新增点坐标
2. `data/stations.json` 新增站点属性
3. `data/status.json` 新增站点状态
4. 更新 `lineStations` 数组

**Q: 告警功能不触发？**
A: 检查 `status.json` 中是否有站点的 `status` 为 `"abnormal"`，且 `message` 不为空。

**Q: 底部卡片显示 N/A？**
A: 当前时间不在 6:00-24:00 范围内，或 `dashboard.json` 文件加载失败。检查控制台是否有网络请求错误。

## 十、项目开发笔记

### 踩过的坑

1. **Cesium Entity 的自定义属性**
   `viewer.entities.add({ stationData: data })` 不会把 `stationData` 真正挂到 Entity 上，因为 Cesium 的 Entity 构造器只处理它已知的属性。必须在 `add()` 之后单独赋值。

2. **Flex 布局中 overflow 失效**
   在 Flex 布局中，子元素的 `overflow-y: auto` 需要配合 `min-height: 0`（或 `overflow: hidden` 的父元素）才能生效。否则 flex 子项的默认 `min-height: auto` 会阻止滚动容器的收缩。

3. **CSS 中 display: none 的过渡**
   详情面板使用 class 切换来控制显隐，而不是 JavaScript 直接操作 style.display，有利于样式和逻辑分离。

4. **跨脚本作用域问题**
   非模块的 `<script>` 标签中，用 `let` 声明的变量只在该脚本块内有效。底部仪表盘脚本最初直接引用了其他脚本的变量，导致 ReferenceError。修复方案是每个脚本独立获取所需数据。

### 后续扩展方向

- 多线路支持：加载多条线路的 GeoJSON 并添加切换 UI
- 后端对接：用 WebSocket 替换 fetch 轮询，实现实时数据推送
- 站点动画：在站点位置添加脉冲圆环（EllipseGraphics）
- 动态模拟：用定时器随机切换站点状态，模拟真实监控场景
